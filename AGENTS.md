# AGENTS.md

## Quick start

```bash
bun install
bun run dev          # dev server on port 3000
bun run build        # vite build + tsc --noEmit
```

## Stack

- **TanStack Start** (SSR React framework with file-based routing, Nitro-powered build)
- **Tailwind CSS v4** (CSS-based config in `src/styles/app.css` — no `tailwind.config`)
- **shadcn/react** (`@shadcn/react`) with **base-nova** style, using `@base-ui/react` as headless primitives
- **Bun** as package manager (`bun.lockb`)
- **TypeScript 6** (beta), strict mode, `"moduleResolution": "Bundler"`

## Path aliases

`~/` maps to `./src/*` (defined in `tsconfig.json` paths, resolved by Vite).

## Generated file — do not edit

`src/routeTree.gen.ts` is auto-generated by TanStack Router from files in `src/routes/`. It is excluded from Prettier. Never edit it manually.

## Routing conventions (TanStack Start)

- File-based routes in `src/routes/`. Each file exports `Route` from `createFileRoute()`.
- Parameter segments: `$paramName` (e.g. `posts.$postId.tsx`).
- Pathless layouts: prefix with `_` (e.g. `_pathlessLayout.tsx`).
- Nested layouts: directory + matching route file (e.g. `posts.tsx` + `posts.$postId.tsx`).
- Index routes: `index.tsx` or `posts.index.tsx`.
- API/server routes: `.ts` files (not `.tsx`) in `src/routes/` — use `server.handlers` on `createFileRoute()` without a `component`.
- Custom extensions in URL: `customScript[.]js.ts` → matches `/customScript.js`.

## Data loading patterns

- **Loader** (`component` routes): `loader` option on `createFileRoute()`. Use `Route.useLoaderData()` in the component.
- **Server functions** (`src/utils/`): `createServerFn()` from `@tanstack/react-start`. Call them directly from components — the framework RPCs them to the server.
- **Middleware** (API routes): `createMiddleware().server(...)`, chain via `.middleware([...])`.

## shadcn component conventions

- Components in `src/components/ui/` — 60 pre-built components.
- Utility: `cn()` from `~/lib/utils` (clsx + tailwind-merge).
- Component variants via `class-variance-authority`.
- The UI is **not** Radix-based — it uses `@base-ui/react` primitives. Look at existing components for pattern reference before adding new ones.

## CSS / theming

- Tailwind v4 `@theme inline` block in `src/styles/app.css` provides shadcn design tokens as Tailwind utilities (e.g. `bg-primary`, `text-muted-foreground`).
- Light/dark via CSS variables and `.dark` class. Theme toggling via `next-themes`.
- Font: Inter Variable (`@fontsource-variable/inter`).

## Project purpose

This is a **starter template** for an AI agency platform (see `PROJECT_PLAN.md`). The intended workflow: an AI coding agent clones this repo, loads relevant skills from `.agents/skills/`, and customizes it into a client-specific website. The repo is designed to be the foundation that a coding agent modifies — not an app that runs on its own.

## Installed agent skills

Skills are tracked in `skills-lock.json` and live in `.agents/skills/`. The coding agent should load relevant skills before modifying the project. Key skill categories: TanStack libraries, Cloudflare platform, frontend design, Vercel React best practices.
